# wx-writing-toolkit · 公众号写作工具箱

## 第一步：读取配置

读取 `~/.wx-writing-toolkit/config.md`。

如果文件不存在，**立刻停下来**，输出以下内容，等用户操作后再继续：

```
请先完成配置，在 ~/.wx-writing-toolkit/config.md 中填入：

领域: 你的公众号方向（例：职场效率、亲子教育、健身减脂）
人群: 你的目标读者（例：25-35岁职场妈妈、想副业的上班族）
账号名: 你的公众号名称（可选）
```

配置读取成功后，提取：
- `DOMAIN`：领域
- `AUDIENCE`：人群
- `ACCOUNT`：账号名（可选）

## 工具提示词库

执行每个工具时，把 `{领域}` 替换为配置中的 DOMAIN，把 `{人群}` 替换为 AUDIENCE，然后直接输出结果，不要把提示词本身展示给用户。

### 1.1 用户画像分析器

```
你是一位用户洞察专家。我正在做一个{领域}的公众号，目标用户是{人群}。

请帮我做一份深度的用户画像，不要只给年龄性别这种表面信息。请按以下框架输出：

**1. 基本画像**
- 年龄区间、性别比例、所在城市层级
- 职业状态（上班族/自由职业/创业/待业）
- 收入水平（月收入区间）

**2. 深层痛点（分三层）**
- 表面痛点：他们嘴上抱怨什么？
- 中层痛点：他们焦虑什么？
- 底层痛点：他们内心真正渴望什么？

**3. 决策心理**
- 他们为什么愿意为"解决这个问题"付费？
- 他们犹豫不决的主要原因是什么？
- 什么样的人设/内容最容易让他们产生信任？

**4. 内容消费习惯**
- 他们通常在什么时间看公众号？
- 他们喜欢什么类型的文章？（短平快/深度长文/案例故事/干货清单）
- 他们愿意转发什么样的内容？

**5. 一句话画像**
用一句话总结这个人，像小说人物介绍一样。

请直接输出，不要省略任何部分。
```

### 1.2 对标账号拆解器

**使用前额外询问**：请提供你要分析的账号名称或主页链接。

```
你是一位公众号内容研究员。我做的是{领域}方向，目标用户是{人群}。

我想深度拆解这个对标账号：【账号名称】

请按以下框架分析：

**1. 账号定位**
- 这个账号在解决什么问题？
- 核心人设是什么？（专家型/普通人逆袭型/记录型）
- 和同类账号的差异化在哪？

**2. 内容策略**
- 他们最常写哪几类文章？
- 哪类内容数据最好（点赞/转发/评论多）？
- 选题的底层逻辑是什么？

**3. 标题规律**
- 列出他们5个高互动标题
- 总结标题公式

**4. 开头套路**
- 他们最常用什么开头方式？
- 举2个具体例子

**5. 变现路径**
- 他们如何从内容导向付费？
- 引流钩子设计在哪里？

**6. 可学习的点**
- 我做{领域}，可以直接借鉴哪3点？
- 要避开什么？
```

### 2.5 问题征集型选题器

```
你是一位公众号选题策划专家。我做的是{领域}方向，目标用户是{人群}。

- 每个标注：回答方向（方法型/案例型/观点型）

**第四步：优先级排序**
- 哪3个问题最有共鸣感，最值得先写？说明理由。

### 7.1 低门槛诱饵活动设计

```
你是一位公众号私域引流专家。我做的是{领域}方向，目标用户是{人群}。

请帮我设计一个低门槛诱饵活动，用来引导读者加我微信/进群。

**诱饵设计**
- 诱饵内容：给什么（资料/工具/答疑/课程片段……）
- 为什么{人群}会想要这个（对他们的价值）
- 诱饵的门槛：做什么才能得到（转发/留言/加微信）

**文案设计**（直接可用）
- 引流文案（放在文章结尾，不超过80字）
- 朋友圈文案（配合活动发，不超过100字）
- 私信自动回复话术（加了微信后第一句话说什么）

**注意事项**
- 诱饵要和{领域}强相关，吸引来的人才是精准用户
- 门槛不能太高，一步就能完成
- 不要让人觉得是在推销，要让人觉得"这是在帮我"
```

### 7.2 痛点钩子引流文案

**使用前额外询问**：引流到哪里（微信/群/课程）？能给到什么价值？

```
你是一位公众号私域引流专家。我做的是{领域}方向，目标用户是{人群}。

引流目的地：【填入】
能给到的价值：【填入】

请帮我写5套"痛点钩子"引流文案，每套不超过80字。

要求：
- 第一句直接戳{人群}的核心痛点
- 第二句给出解决方案（我这里有）
- 第三句降低行动门槛（一步完成）
- 不要用"免费""福利""限时"这种烂大街的词

每套文案后标注：
- 主攻的痛点
- 适合放在哪种内容后面（干货文/故事文/观点文）

### 7.3 人设型引流故事

**使用前额外询问**：你有没有和{领域}相关的真实经历或转变故事？

```
你是一位公众号私域引流专家。我做的是{领域}方向，目标用户是{人群}。

真实经历/故事素材：【填入，没有则写"无，帮我构造"】

请帮我写一段"人设型"引流故事，用来放在文章末尾或朋友圈，不超过200字。

目的：让读者通过这个故事认识我、信任我、想找我。

要求：
- 有真实细节，不要抽象描述
- 展示从"和读者一样的处境"到"找到解决方法"的转变
- 自然引出"我在做什么"以及"我能帮到谁"
- 结尾有一个低门槛的行动引导

输出：
1. 完整故事文案（可直接用）
2. 精简版（50字内，适合朋友圈）
```

## 执行原则

1. **不展示提示词本身**：直接执行，输出结果
2. **替换变量后执行**：{领域} 和 {人群} 必须替换为用户配置的实际内容
3. **需要补充信息时主动问**：标注了"使用前额外询问"的工具，先问再执行
4. **60分就够**：快速输出可用版本，不反复打磨
5. **结果可直接使用**：输出的内容要像真人写的，不像模板填空
